# 娱乐 Agent · 手机端 H5 · 产品架构文档

> 版本：v2.0 | 更新：2026-03-29 | Owner：Miles Zhang

---

## 1 项目定位

手机端作为遥控器，向车端 AI 大脑发送用户意图 JSON，车端推理后编排桌面卡片。

| 场景 | 用户行为 | 车端预期 |
|------|---------|---------|
| 娱乐需求响应 | 输入"不想听音乐了，我想听播客" | 喜马拉雅推荐卡移入堆栈顶 |
| 上车智能推荐 | 选择用户 + 行程 + 触发上车 | 根据 persona × trip × time 推荐最佳卡片 |
| AI 播客生成 | 发送内容链接 | 生成播客 → 播客卡移入堆栈顶 |

## 2 系统架构

```
┌─────────────┐          ┌──────────────────────┐          ┌──────────┐
│  手机端 H5   │←──WSS──→│   阿里云服务器         │←──WSS──→│  车端 AI  │
│  Netlify     │          │                      │          │  Android  │
└─────────────┘          │  relay.js (:9000)     │          └──────────┘
       │                 │  podcast-api.js(:9001)│
       │── POST ────────→│  Nginx (:8443 SSL)    │
       │  /api/podcast   │  audio/ (mp3 存储)    │
       │                 │  podcast-store.json   │
       │                 └──────────────────────┘
       │                          │
       │                          │── WSS + Header ──→ 豆包 API
       │                          │   (播客生成代理)
```

| 服务 | URL |
|------|-----|
| 手机端 H5 | https://stalwart-sunburst-2b6e9f.netlify.app/ |
| 中继 WSS | wss://zhangchang.duckdns.org:8443/ws?room=car_001 |

### 各端职责

| 端 | 负责 | 不负责 |
|----|------|-------|
| **手机端** | 用户切换、文字输入、播客触发、上车触发、JSON 发送 | 推理、渲染、播放 |
| **服务器** | 消息中继、播客生成代理、音频存储 | 业务决策 |
| **车端** | 接收 JSON → LLM 推理 → 卡片渲染 → 音频播放 → ACK | 用户输入 |

## 3 AI 播客链路

### 播客生成流程

```
手机输入 URL → POST /api/podcast
  │
  ├─ store 中已有？ → YES → 直接返回缓存（0 token，0 等待）
  │
  NO → 豆包 API 生成（2-10 分钟）
  │    → SSE 流式进度推送给手机端
  │    → PodcastEnd 拿到 cdn_url
  │    → 下载 mp3 到服务端持久存储
  │    → 写入 podcast-store.json
  │
  └→ 手机端组装 postcard JSON → 经中继发送到车端 → 车端播放
```

### 三阶段 UI 状态

| 阶段 | 触发 | 手机端显示 |
|------|------|-----------|
| 1 | 用户发出 URL | "链接已收到！正在生成播客..." |
| 2 | 豆包开始生成 | "正在生成第 N 段..." |
| 3 | 生成完成 | "生成完成！请在车端查看播客详情" |

缓存命中时直接跳到阶段 3（"播客已就绪！"）。

### 预设播客 vs 实时生成

| | 预设播客 | 实时生成 | 缓存命中 |
|--|--------|---------|---------|
| 触发 | 从预设列表选择 | 输入自定义 URL | 输入已有 URL |
| 等待 | 0s | 2-10 分钟 | 0s |
| Token 消耗 | 0 | ~15K-50K | 0 |
| JSON type | `source.type: "preset"` | `source.type: "url"` | `source.type: "url"` |

### 性能基线

| 指标 | 值 |
|------|-----|
| 生成速度 | 2.7× 实时（1 分钟播客只需 22s） |
| 首 token | avg 6s |
| 音频格式 | MP3, 96kbps, 24kHz mono |

## 4 六条交互链路

### A 切换用户

点击顶栏用户胶囊 → 底部弹窗选择 → 更新本地状态（不发 JSON）。

### B 文字输入 → content JSON

输入文字 → 发送 → 用户气泡 + content 反馈卡：
- 文案：「您的需求已收到！将在车端显示效果」
- 底部状态：发送中 → 已发送至车端 / 发送失败（含原因 + 重试按钮）

### C 触发上车 → recommend JSON

点击「模拟上车」→ 弹窗选行程 → 系统消息 + recommend 反馈卡：
- 文案：「用户上车了！需要推荐适合用户和当前场景的最佳内容」
- 标签行：用户标签 + 场景标签 + 时长标签

### D+E AI 播客 → postcard JSON

点击「AI播客」→ 弹窗选预设/输入 URL → 生成状态 + postcard 反馈卡：
- 标题：文章真实标题（服务端自动提取）
- 副标题：「两位AI主播为你深度解读这篇文章，涵盖核心观点和延伸信息」
- 原链接：可点击跳转
- 时长：`时长 mm:ss · MP3 · 96kbps`

### F 查看 JSON

点击任意卡片「查看JSON」→ 弹窗展示格式化 JSON。

## 5 播客内容存储

### 存储位置

```
/opt/relay/
├── podcast-store.json     # 播客索引
└── audio/                 # mp3 文件
```

### 数据模型

```json
{
  "id": "gen_1774767112003",
  "source_url": "https://mp.weixin.qq.com/s/xxx",
  "title": "小米在人工智能和机器人领域",
  "desc": "小米最近的AI技术进展...",
  "audio_file": "gen_1774767112003.mp3",
  "duration_sec": 682,
  "created_at": "2026-03-29T14:00:00Z",
  "source": "preset | generated"
}
```

### 去重机制

同一 `source_url` 只生成一次。后续请求直接返回缓存。

### 音频持久化

豆包 CDN URL 24h 过期。服务端在生成完成后立即下载 mp3 到本地，车端拿到的是服务端永久 URL。

## 6 部署架构

### 服务器（阿里云）

| 组件 | 端口 | 说明 |
|------|------|------|
| Nginx | 8443 (SSL) | Let's Encrypt 证书，反代所有服务 |
| relay.js | 9000 | WebSocket 房间广播 |
| podcast-api.js | 9001 | 播客生成 / 存储 / 音频服务 |
| PM2 | — | 进程守护 + 开机自启 |

### Nginx 路由

| 路径 | 目标 |
|------|------|
| `/ws` | relay.js（WebSocket 升级） |
| `/api/podcast` | podcast-api.js/generate（SSE） |
| `/api/podcast/list` | podcast-api.js/list |
| `/api/podcast/audio/` | podcast-api.js/audio/ |

### 部署命令

```bash
SSHPASS=密码 ./server/deploy.sh   # 只更新代码，不碰数据
```
