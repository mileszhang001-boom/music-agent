# 车端 Android 对接指南

> 版本：v2.0 | 更新：2026-03-29

---

## 一句话理解

车端通过 WebSocket 连接中继服务器，接收手机端发来的 JSON → LLM 推理 → 渲染卡片 → 回传 ACK。

```
手机 H5 (Netlify) ←─WSS─→ 中继服务器 (阿里云) ←─WSS─→ 车端 Android
```

---

## 1. 连接方式

### WebSocket 地址（APK 中硬编码）

```
wss://zhangchang.duckdns.org:8443/ws?room=car_001&role=car
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 协议 | `wss://` | Let's Encrypt 正式证书 |
| 端口 | `8443` | 非标端口 |
| 路径 | `/ws` | Nginx 反代到 relay 服务 |
| `room` | `car_001` | 房间号，手机和车端必须相同 |
| `role` | `car` | 标识身份 |

### Android 代码示例

```kotlin
val client = OkHttpClient.Builder()
    .readTimeout(0, TimeUnit.MILLISECONDS)
    .build()

val request = Request.Builder()
    .url("wss://zhangchang.duckdns.org:8443/ws?room=car_001&role=car")
    .build()

client.newWebSocket(request, object : WebSocketListener() {
    override fun onMessage(webSocket: WebSocket, text: String) {
        val json = JSONObject(text)

        // ⚠️ 必须过滤中继系统事件
        if (json.optString("type") == "_relay_event") return

        // 处理业务消息
        handleMessage(json, webSocket)
    }
})
```

### 断线重连（必须实现）

中继不保证永不断连。车端必须实现重连：

```kotlin
var retryDelay = 1000L
val maxDelay = 10000L

fun reconnect() {
    handler.postDelayed({
        connectWebSocket()
        retryDelay = min(retryDelay * 2, maxDelay)
    }, retryDelay)
}

// 连接成功后重置
override fun onOpen(...) {
    retryDelay = 1000L
}
```

## 2. 消息协议

### 通用信封

所有业务消息共用一个顶层结构：

```json
{
  "type": "content | recommend | postcard",
  "session_id": "sess_20260329_103000",
  "timestamp": "2026-03-29T10:30:00.000Z",
  "user_profile": {
    "persona_id": "user_a",
    "persona_label": "欧美流行重度发烧友"
  },
  "payload": { ... }
}
```

**⚠️ 关键**：`type` 为 `_relay_event` 的消息是中继系统事件，**不是业务消息，必须跳过**。

---

### ① content — 用户说了一句话

**车端该做什么**：根据 `text` + `user_profile` 推理，决定推哪张卡到堆栈顶。

```json
{
  "type": "content",
  "session_id": "sess_20260329_103000",
  "timestamp": "2026-03-29T10:30:00.000Z",
  "user_profile": { "persona_id": "user_a", "persona_label": "欧美流行重度发烧友" },
  "payload": {
    "text": "不想听音乐了，我想听播客"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `payload.text` | string | 用户原文 |

**演示测试用例**：

| 输入 | 期望车端行为 |
|------|-------------|
| "不想听音乐了，我想听播客" | 喜马拉雅推荐卡 → 堆栈顶 |
| "导航回家" | 地图卡 → 堆栈顶 |
| "太热了" | 空调控制卡 → 堆栈顶 |

---

### ② recommend — 用户上车了

**车端该做什么**：根据 `user_profile` + `trip` + `time_context` 做全量卡片编排。

```json
{
  "type": "recommend",
  "session_id": "sess_20260329_103500",
  "timestamp": "2026-03-29T10:35:00.000Z",
  "user_profile": { "persona_id": "user_b", "persona_label": "喜欢听国语民谣、轻音乐" },
  "payload": {
    "trigger": "user_enter_car",
    "trip": {
      "scene_id": "morning_commute",
      "scene_label": "早上通勤",
      "duration_min": 30
    },
    "time_context": {
      "hour": 8,
      "day_type": "weekday",
      "period": "early_morning"
    }
  }
}
```

| 字段 | 类型 | 可选值 |
|------|------|--------|
| `trip.scene_id` | string | `morning_commute` / `commute_home` / `weekend_outing` / `couple_time` / `kid_mode` |
| `trip.duration_min` | number | 用户输入，通常 15-120 |
| `time_context.hour` | number | 0-23（自动获取） |
| `time_context.day_type` | string | `weekday` / `weekend` |
| `time_context.period` | string | `early_morning`(5-8) / `morning`(9-11) / `afternoon`(12-16) / `evening`(17-20) / `late_night`(21-4) |

---

### ③ postcard — AI 播客可以播了

**车端该做什么**：将播客卡插入堆栈顶，用 `cdn_url` 播放音频。

```json
{
  "type": "postcard",
  "session_id": "sess_20260329_104000",
  "timestamp": "2026-03-29T10:40:00.000Z",
  "user_profile": { "persona_id": "user_c", "persona_label": "喜欢听古典乐、播客" },
  "payload": {
    "source": {
      "type": "preset",
      "url": "https://mp.weixin.qq.com/s/g5-Y-7H1hfovmyBcB6WSqQ",
      "title": "AI 重塑未来工作方式"
    },
    "podcast": {
      "cdn_url": "https://zhangchang.duckdns.org:8443/api/podcast/audio/ai_tech_01.mp3",
      "duration_sec": 1110,
      "format": "mp3",
      "speakers": [
        { "id": "zh_male_dayixiansheng_v2_saturn_bigtts", "role": "host_a" },
        { "id": "zh_female_mizaitongxue_v2_saturn_bigtts", "role": "host_b" }
      ]
    }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `source.type` | string | `"preset"` 预设 / `"url"` 用户输入链接 |
| `source.title` | string | 播客标题 |
| `podcast.cdn_url` | string | **直接可播放的 MP3 URL** |
| `podcast.duration_sec` | number | 总时长（秒） |

## 3. ACK 回传

**收到任何业务消息后，处理完毕立即回传**：

```json
{
  "status": "ok",
  "session_id": "sess_20260329_103000",
  "message": "推荐已更新"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | `"ok"` 成功 / `"error"` 失败 |
| `session_id` | string | **必须与收到消息的 session_id 一致** |
| `message` | string | 可选描述 |

通过同一个 WebSocket 连接发送，中继自动转发给手机端。

**手机端超时**：10 秒内未收到 ACK → 显示"发送失败，车端未响应"。如果 LLM 推理超过 10s，建议先回一个 `{"status":"ok"}` 表示已收到。

## 4. 音频播放

### 播放方式

Android MediaPlayer 支持 HTTP URL 边下边播：

```kotlin
val mediaPlayer = MediaPlayer().apply {
    setDataSource(cdnUrl)  // postcard.podcast.cdn_url
    prepareAsync()
    setOnPreparedListener { start() }
}
```

### 音频规格

MP3, 96kbps, 24kHz, Mono。Android MediaPlayer 直接支持。

### cdn_url 有效期

所有播客均已下载到服务端存储，URL 格式为 `https://zhangchang.duckdns.org:8443/api/podcast/audio/xxx.mp3`，**永久有效**。

## 5. 中继系统事件

连接后会收到 `type: "_relay_event"` 的消息。**这不是业务消息，必须跳过**（不要传给 LLM 推理）。

```kotlin
// 最简处理：直接跳过
if (json.optString("type") == "_relay_event") return
```

如果车端**想要**显示手机连接状态（可选，不是必须），可以读取以下事件：

| event | 说明 |
|-------|------|
| `peer_joined` | 手机端上线 |
| `peer_left` | 手机端离线 |

## 6. 常见问题

| 问题 | 回答 |
|------|------|
| 断连后消息会丢吗？ | 会。中继无持久化，重连后由用户手动重发 |
| 多台手机连同一 room？ | 所有消息都到达车端，通过 `persona_id` 区分 |
| 需要处理所有三种消息？ | 是。`content`/`recommend` 需推理，`postcard` 直接播放，三种都回 ACK |
| ACK 必须立即回传？ | 10s 超时。推理慢的话先回 ok，再渲染 |
| cdn_url 会过期吗？ | 不会，已持久化到服务端 |

```bash
npm install -g wscat

# 模拟车端
wscat -c "wss://zhangchang.duckdns.org:8443/ws?room=car_001&role=car"

# 连接后会收到:
# {"type":"_relay_event","event":"connected","room":"car_001",...}

# 手机端操作后，这里收到 JSON
# 手动回复 ACK:
{"status":"ok","session_id":"收到消息中的session_id","message":"已处理"}
```
