### 2.4 postcard（实时生成）— 完整链路

```
手机                   Nginx:8080       podcast-api:9001     豆包 API        relay:9000       车端
 │                       │                  │                  │                │               │
 │  输入 URL             │                  │                  │                │               │
 │                       │                  │                  │                │               │
 │── POST /api/podcast ─→│── proxy ────────→│                  │                │               │
 │   {url: "https://.."} │                  │                  │                │               │
 │                       │                  │── 查 store 去重   │                │               │
 │                       │                  │  (命中→直接返回)   │                │               │
 │                       │                  │                  │                │               │
 │← SSE: phase 1 ───────│←─────────────────│                  │                │               │
 │  "链接已收到！"        │                  │                  │                │               │
 │                       │                  │── 抓取文章标题 ───→│(fetch URL)     │               │
 │← SSE: meta ──────────│←─────────────────│← title/desc ─────│                │               │
 │                       │                  │                  │                │               │
 │                       │                  │── WSS 建连 ──────────────────────→│               │
 │                       │                  │   (带4个鉴权Header)               │               │
 │                       │                  │← ConnectionStarted ──────────────│               │
 │                       │                  │── StartSession ─────────────────→│               │
 │                       │                  │                  │                │               │
 │← SSE: phase 2 ───────│←─────────────────│                  │                │               │
 │  "已开始生成"          │                  │                  │                │               │
 │                       │                  │  ┌─ 流式循环 ────┐│                │               │
 │← SSE: progress ──────│←─────────────────│←─┤ RoundStart   ││                │               │
 │  "第N段..."           │                  │  │ RoundResp    ││                │               │
 │                       │                  │  └──────────────┘│                │               │
 │                       │                  │                  │                │               │
 │                       │                  │← PodcastEnd ─────│                │               │
 │                       │                  │  {audio_url}     │                │               │
 │                       │                  │                  │                │               │
 │                       │                  │── 下载 mp3 到本地  │                │               │
 │                       │                  │── 写入 store      │                │               │
 │                       │                  │                  │                │               │
 │← SSE: complete ──────│←─────────────────│                  │                │               │
 │  {audioUrl, title,    │                  │                  │                │               │
 │   durationSec}        │                  │                  │                │               │
 │                       │                  │                  │                │               │
 │── ws.send(postcard) ─→│── proxy /ws ───────────────────────────────────────→│── broadcast ─→│
 │                       │                  │                  │                │               │
 │                       │                  │                  │                │  播放 cdn_url  │
 │                       │                  │                  │                │               │
 │←── ACK ──────────────│←─────────────────────────────────────────────────────│←── ACK ──────│
```

## 3. 超时与重试机制

### 超时值一览

| 环节 | 超时 | 位置 | 超时后行为 |
|------|------|------|-----------|
| ACK 等待 | **10s** | api.js | 卡片显示"发送失败，车端未响应" + 重试按钮 |
| 播客生成 | **900s** (15min) | podcast-api.js | SSE error "生成超时" |
| 文章标题抓取 | **10s** | podcast-api.js | fallback 用豆包文案提取 |
| 音频下载 | **120s** | podcast-api.js | 下载失败，但 store 仍写入 |
| Nginx WS 空闲 | **86400s** (24h) | Nginx | 连接断开 |
| Nginx SSE | **900s** | Nginx | 代理超时 |

### 重连策略

```
断开 → 1s 后重连 → 失败 → 1.5s → 2.25s → 3.4s → ... → 最大 10s
```

指数退避，乘数 1.5，上限 10s。连接成功后重置。

### 重试按钮

- 仅在 `ackStatus === 'error'` 时显示
- 重试 = 重发同一份 JSON，**不重新生成播客**
- 播客 cdn_url 已在首次生成时持久化

## 4. 内容存储

### 存储位置

```
/opt/relay/
├── podcast-store.json    # 索引（当前 9 条）
└── audio/                # mp3 文件
    ├── ai_tech_01.mp3    # 预设 × 7
    ├── gen_xxx.mp3       # 用户生成 × 2
    └── ...
```

### 去重逻辑

```
POST /api/podcast {url}
  → findByUrl(url) in store
  → 命中: SSE complete(cached:true)，0 token，0 等待
  → 未命中: 调豆包 → 下载 → 入库
```

### 数据模型

```json
{
  "id": "gen_1774767112003",
  "source_url": "https://mp.weixin.qq.com/s/xxx",
  "title": "文章主题（自动提取）",
  "desc": "文章描述",
  "audio_file": "gen_1774767112003.mp3",
  "duration_sec": 682,
  "created_at": "2026-03-29T14:00:00Z",
  "source": "generated"
}
```

## 5. 关键修复记录

| 日期 | 问题 | 原因 | 修复 |
|------|------|------|------|
| 03-29 | 阿里云 80/443 端口不可用 | 未备案域名 ICP 拦截 | 改用 8443 非标端口 |
| 03-31 | WSS 全端口不可用 | 阿里云拦截 TLS 握手（SNI 阻断） | 改用 HTTP 8080 纯明文 |
| 03-31 | 车端收不到消息 | relay.js 发 binary frame，OkHttp 只收 text frame | `broadcast(room, ws, msgStr)` |
| 03-29 | duration_sec 返回 0 | 豆包 PodcastEnd 不总返回此字段 | 音频大小估算 (96kbps=12KB/s) |
| 03-29 | 微信标题抓取失败 | 服务器 IP 被微信反爬（"环境异常"） | 用豆包 Round 2 文案提取 |
| 04-03 | podcast-api 崩溃 | 端口 9001 被残留进程占用 (EADDRINUSE) | kill 残留 + pm2 restart |
