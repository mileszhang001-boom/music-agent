## 整体架构

```
┌──────────────┐     HTTP      ┌──────────────────────────────────────┐     WS      ┌─────────────┐
│  手机浏览器   │──── :8080 ───→│          阿里云 47.94.241.139         │←── :9000 ──→│  车端 Android │
│              │               │                                      │             │  OkHttp WS   │
└──────────────┘               │  Nginx (:8080)                       │             └─────────────┘
                               │    ├── /        → H5 静态文件         │
                               │    ├── /ws      → relay.js (:9000)   │
                               │    ├── /api/podcast → podcast-api.js │
                               │    └── /health  → 200 OK             │
                               │                                      │
                               │  relay.js (:9000)                    │
                               │    WebSocket 房间广播                  │
                               │                                      │
                               │  podcast-api.js (:9001)              │
                               │    播客生成 / 存储 / 音频服务           │
                               │                                      │
                               │  PM2 进程守护                         │
                               └──────────────────────────────────────┘
                                          │
                                          │ WSS + 鉴权 Header
                                          ↓
                                   豆包 TTS API
                                   (播客生成时调用)
```

## 端口与进程

| 进程 | 端口 | 协议 | 说明 |
|------|------|------|------|
| **Nginx** | 8080 | HTTP | 入口，反代所有服务。手机浏览器直接访问此端口 |
| **relay.js** | 9000 | WS | WebSocket 房间广播。Nginx `/ws` 代理到此 |
| **podcast-api.js** | 9001 | HTTP | 播客生成 API。Nginx `/api/podcast*` 代理到此 |
| **Nginx** | 8443 | HTTPS/WSS | SSL 版本（当前因 SNI 阻断暂不使用） |

车端 Android 可直连 `ws://47.94.241.139:9000` 或通过 Nginx `ws://47.94.241.139:8080/ws`。

## 链路 1：文字输入（content）

```
手机输入 "我想听播客"
    ↓
main.js: buildContent(text) → JSON
    ↓
api.js: ws.send(JSON) → Nginx :8080/ws → relay.js :9000
    ↓
relay.js: broadcast(msgStr) → 车端 WS 连接
    ↓
车端: onMessage(text) → LLM 推理 → 渲染卡片
    ↓
车端: ws.send(ACK JSON) → relay.js → 手机端
    ↓
手机端: 卡片状态更新 "已发送至车端"
```

## 链路 2：上车推荐（recommend）

```
手机选择 "早上通勤 30min"
    ↓
main.js: buildRecommend(scene, duration) → JSON（含 time_context 自动计算）
    ↓
api.js → relay.js → 车端（同链路 1）
    ↓
车端: 根据 persona + trip + time 编排全量卡片 → ACK
```

## 链路 3：AI 播客（postcard）

### 预设播客（0s 等待）

```
手机选择预设卡片
    ↓
main.js: buildPostcard(preset.cdn_url) → JSON
    ↓
api.js → relay.js → 车端
    ↓
车端: MediaPlayer.setDataSource(cdn_url) → 播放 → ACK
```

### 实时生成（2-10 分钟）

```
手机输入 URL
    ↓
podcast.js: fetch POST → Nginx :8080/api/podcast → podcast-api.js :9001
    ↓
podcast-api.js:
  1. 查 store 去重（已有 → 直接返回，0 token）
  2. 抓取文章标题（fetch → og:title）
  3. 连接豆包 WSS（带鉴权 Header）
  4. 流式接收 → SSE 推送进度给手机
  5. PodcastEnd → 下载 mp3 到 /opt/relay/audio/
  6. 写入 podcast-store.json
  7. 返回 {audioUrl, title, durationSec}
    ↓
手机端: buildPostcard(audioUrl) → JSON → relay.js → 车端
    ↓
车端: MediaPlayer 播放 → ACK
```

## 链路 4：ACK 回传

```
车端处理完毕
    ↓
车端: ws.send({"status":"ok","session_id":"xxx"})
    ↓
relay.js: broadcast(msgStr) → 手机端
    ↓
手机端: pendingAcks.get(session_id).resolve(ack)
    ↓
卡片状态: "发送中..." → "已发送至车端"
```

超时（10s 无 ACK）→ "发送失败，车端未响应" + 重试按钮

## 关键修复记录

### text frame vs binary frame（2026-03-31）

**问题**：车端 Android (OkHttp) 收不到中继转发的消息。

**原因**：Node.js `ws` 库收到 WebSocket 消息后 `data` 类型是 `Buffer`。调用 `ws.send(Buffer)` 时发送的是 **binary frame**。但 Android OkHttp 的 `onMessage(String text)` 回调只接收 **text frame**，不触发 binary frame。

**修复**：`relay.js` 第 115 行，将 `broadcast(room, ws, data)` 改为 `broadcast(room, ws, msgStr)`，其中 `msgStr = data.toString()`。

**影响**：所有消息（JSON 文本）通过 text frame 转发，Android OkHttp 正常接收。

### SNI/TLS 阻断（2026-03-31）

**问题**：阿里云对未备案域名（duckdns.org）拦截 TLS 握手，WSS 不可用。

**方案**：新增 Nginx 8080 端口纯 HTTP 通道，H5 和 WS 均走明文。8443 SSL 通道保留但暂不使用。

## 文件清单

### 服务器 `/opt/relay/`

| 文件 | 说明 |
|------|------|
| `relay.js` | WebSocket 中继 + HTTP 诊断接口 |
| `podcast-api.js` | 播客生成 / 存储 / 音频服务 |
| `ecosystem.config.cjs` | PM2 配置（含豆包 API 凭证） |
| `podcast-store.json` | 播客索引（9 条：7 预设 + 2 生成） |
| `audio/` | mp3 文件（9 个） |
| `h5/` | H5 构建产物（Vite dist） |

### Nginx `/etc/nginx/sites-enabled/`

| 文件 | 端口 | 说明 |
|------|------|------|
| `relay-http` | 8080 | 当前使用。HTTP 入口 |
| `relay` | 8443 | 备用。SSL 入口（SNI 阻断暂不用） |

### 诊断工具

| 端点 | 说明 |
|------|------|
| `GET http://localhost:9000/` | 房间状态（连接数、IP、消息计数） |
| `GET http://localhost:9000/test/car_001` | 向房间发测试消息 |
| `pm2 logs relay` | 实时日志（含 ✅ ACK 标记） |
